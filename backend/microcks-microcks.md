### 中文

**项目简介**  
Microcks（microcks/microcks）是一款开源、云原生的 API Mock 与测试工具，已入选 CNCF Incubating 项目。它帮助团队在不重新搭建通用后端组件的前提下，快速复用已有的服务基础设施，实现 API 的快速交付与高质量验证。

**价值主张**  
- **加速 API 交付**：通过自动生成 Mock 服务和集成测试，显著缩短开发、调试和上线的周期。  
- **复用后端基础设施**：统一的 Mock 与测试平台让不同团队共享同一套服务契约，避免重复建设。  
- **标准化服务模式**：提供统一的 API 描述（OpenAPI、AsyncAPI、GraphQL 等）和验证规则，提升跨团队协作的一致性。

**典型接入方式**  
1. **Docker / Kubernetes 部署**：官方提供 Docker 镜像，可直接在本地 Docker、Docker‑Compose 或 K8s（Helm Chart）中启动。  
2. **CLI / SDK 集成**：通过 `microcks-cli` 或 Java/Node 等语言的 SDK，将 Mock 注册、导入契约、触发测试等操作脚本化，便于 CI/CD 流程自动化。  
3. **CI/CD 插件**：支持在 Jenkins、GitLab CI、GitHub Actions 等平台上调用 Microcks REST API，实现每次代码提交后自动校验 API 合规性。

- **生态兼容**：提供丰富的 Helm Chart、Docker 镜像以及 REST/CLI 接口，易于与 Service Mesh、Istio、Kong 等云原生组件集成。
