### 中文

**项目简介（2‑3 句）**  
NativeScript 是一个基于 TypeScript 的跨平台移动开发框架，支持使用 Angular、React、Vue、Svelte、Solid 等前端框架，同时可以直接调用 iOS（UIKit、SwiftUI）和 Android（View、Jetpack Compose）原生 API，甚至兼容 Flutter、Dart 等生态。它让前端团队能够用熟悉的语言和工具一次编写代码，生成原生 iOS 与 Android 应用。

**价值**  
- **快速交付 UI**：复用已有的前端组件库，省去大量原生 UI 的手写工作。  
- **统一技术栈**：前端团队只需掌握 TypeScript 与常用前端框架，即可同时构建 Web 与移动端界面。  
- **原生性能**：通过直接调用原生控件，保持与原生应用相当的流畅度和用户体验。

**典型接入方式**  
1. **创建项目**：使用 `ns create my-app --template @nativescript/template-blank-react`（或 `angular`、`vue`、`svelte` 等）快速生成带有所选框架的项目。  
2. **编写 UI**：在 `src/app` 中使用框架的 JSX/TSX、模板或 Svelte 语法，配合 `tns-core-modules`（或 `@nativescript/core`）提供的原生 UI 组件。  
3. **平台构建**：通过 `ns run ios` / `ns run android` 完成本地编译，或使用 CI/CD 将产物打包成 IPA / APK。  
4. **渐进式迁移**：在已有移动项目中，可先在一个模块或页面引入 NativeScript，验证兼容性后逐步扩展。

**生产可用性**  
- **活跃度**：截至 2026‑07‑02，项目拥有 25.5k+ ⭐、1.7k+ 🍴，最近一次提交在数天前，表明社区和维护者仍然活跃。  
- **生态成熟**：提供完整的插件市场（如摄像头、文件系统、数据库等），并支持主流前端框架的官方模板。  
- **风险**：需进一步审查许可证（MIT）与安全报告，确认无重大漏洞后方可正式上线。  
- **推荐做法**：先在内部做一个小型 PoC（如登录页或列表页），验证与现有后端、CI 流程的兼容性，再逐步推广至完整产品。整体来看，NativeScript 已具备在生产环境中大规模使用的技术和社区基础。
