**Project Overview**

The stripe/openapi project provides an OpenAPI specification for the Stripe API, enabling faster integration of monetization, billing, and payment service provider (PSP) flows into applications. This open-source project facilitates the integration of billing or checkout processes, evaluation of PSP flows, and automation of payment operations.

**Value Proposition**

The value of stripe/openapi lies in its ability to simplify the integration process, allowing developers to focus on building their applications rather than manually configuring payment gateways. By leveraging this OpenAPI specification, developers can integrate Stripe's payment services more efficiently and effectively.

### 中文

**价值**  
stripe/openapi 提供了 Stripe API 的完整 OpenAPI 规范，开发者可以直接基于该规范生成客户端代码、自动化文档或进行接口 Mock，从而大幅缩短支付、计费、结算等业务的集成时间，尤其适合需要快速验证 PSP（Payment Service Provider）流程或搭建内部原型的团队。

3. 在项目中引入生成的代码，结合自己的业务逻辑完成鉴权（API‑Key、OAuth）和错误处理。  
4. 如需自定义或扩展，可手动编辑规范或在生成后添加业务层包装。
