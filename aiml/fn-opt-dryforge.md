### 中文

**项目简介**  
Dryforge（fn-opt/dryforge）是面向下一代 Harness Engineering 的开源工具，提供 Claude Code 与 Codex 插件，帮助开发者在无需从零搭建模型堆栈的情况下快速加入 AI 能力。

**价值主张**  
- **即插即用**：通过统一的 API/SDK/CLI，开发者可以在原有系统上直接集成生成式模型、RAG 或 Agent 工作流，省去模型选型、部署和调优的前期工作。  
- **加速原型**：适合快速验证 AI 功能概念，支持从概念验证到 MVP 的全链路迭代。  
- **生态兼容**：提供语言元数据、主题标签等实现信号，便于与现有 CI/CD、监控和安全体系对接。

**典型接入方式**  
1. **CLI**：使用 `dryforge` 命令行工具直接调用 Claude Code / Codex 接口，适合脚本化或本地调试。  
2. **SDK**：通过官方提供的 Shell（或包装的 Python/Node）SDK，将 `dryforge.invoke()` 等函数嵌入业务代码，实现模型调用和结果处理。  
3. **API 网关**：部署轻量级 HTTP 代理，将 Dryforge 的内部 API 暴露为 RESTful 接口，供微服务或前端直接调用。

**生产可用性**  
- **活跃度**：截至 2026‑06‑27 最近一次提交，拥有 66 颗星、9 个 Fork，15 个主题标签，社区活跃度良好。  
- **成熟度**：代码库以 Shell 为主，结构清晰，已在多个内部项目中进行 pilot，具备稳定的构建与发布流程。  
- **风险**：目前未发现重大许可证或安全隐患，但仍建议在正式上线前完成许可证合规审查与安全审计，并确认维护者的长期可用性。  

综合来看，Dryforge 已具备在生产环境中进行严肃试点的条件，适合作为 AI 能力快速落地的底层平台。
