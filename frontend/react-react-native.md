### 中文

**项目简介**  
React / React‑Native 是一个基于 React 的跨平台框架，能够使用同一套 JavaScript/JSX 代码同时构建 Web 与原生移动应用的 UI。它通过抽象原生控件，让开发者以声明式方式快速搭建用户界面，显著降低自定义 UI 的工作量。

**价值**  
- **加速 UI 开发**：复用已有的 React 组件，快速交付产品界面。  
- **统一技术栈**：前端团队可使用熟悉的 React 生态，在 iOS、Android 以及 Web 上共享代码。  
- **提升交付效率**：组件化、热更新与社区丰富的插件生态，使前端交付周期更短、质量更可控。

**典型接入方式**  
1. **先行 PoC**：在现有项目中创建一个最小化的 React‑Native 示例（如仅包含一个登录页），验证编译、运行及与原生模块的集成。  
2. **参考官方 README**：按照文档完成 `react-native init`、依赖安装（Node、Yarn、Android SDK / Xcode）以及基本的 iOS/Android 配置。  
3. **逐步迁移**：在 PoC 验证通过后，将业务组件逐步迁入 React‑Native，保持原生代码与 React‑Native 代码并存，利用桥接（Bridge）实现功能互通。  

**生产可用性**  
- **成熟度高**：GitHub ★126k、Fork ★25k，活跃度截至 2026‑07‑01，社区生态完善。  
- **稳定性**：官方发布周期短，持续更新，已被大量企业级产品采用，具备正式生产环境的支撑能力。  
- **风险提示**：元数据未直接提供完整的集成指南，建议在正式投入前通过小规模 PoC 验证搭建成本与团队学习曲线。  

总体而言，React / React‑Native 具备高生产就绪度，适合作为前端团队加速移动端 UI 开发的核心技术栈。
